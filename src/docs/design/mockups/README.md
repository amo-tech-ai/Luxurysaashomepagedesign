## Design Principles (Non-Negotiable)

Every mockup must adhere to:

### Visual Hierarchy
* One primary message per section
* Spacing creates hierarchy (not color or borders)
* Editorial typography leads

### Diagram-First
* Diagrams explain strategy (not decoration)
* Text + lines only
* No photography, illustrations, or decorative icons

### Consulting-Grade Aesthetic
* Feels like BCG/McKinsey partner deck
* Calm, authoritative, precise
* No SaaS marketing tropes

### Content Tone
* Strategic clarity over promotional hype
* Sentence case headlines
* Analytical language

## Mockup Format

Each mockup follows this template:

```markdown
# [Section Name]

## Purpose
What this section accomplishes

## Content Specifications
- Headline
- Subheadline (if applicable)
- Body copy
- Diagram labels
- CTAs

## Visual Specifications
- Background color
- Typography scale
- Spacing (padding, margins)
- Grid structure
- Diagram type

## Layout
### Desktop (1440px max-width)
[Description of layout]

### Mobile (375px–768px)
[Description of responsive behavior]

## Diagram Details
- Type: [Flow/Framework/Layer/Matrix/Stats]
- Structure: [Description]
- Key elements: [Labels, connectors, etc.]

## Implementation Notes
- Technical considerations
- Interaction states
- Accessibility requirements

## References
- BCG inspiration: [If applicable]
- StartupAI page: [Implementation file]
```
