**Project:** StartupAI Fashion Intelligence Platform  
**Purpose:** Reusable design system for premium consulting-grade infographics  
**Aesthetic:** BCG consulting presentation style, luxury, intelligent, calm  
**Last Updated:** February 14, 2026

---

## Philosophy: "Thinking Visuals" Approach

### Core Principles

**Intelligence Over Decoration**
Every visual element must communicate strategic insight. Avoid decorative graphics that do not convey meaning. Diagrams should reveal structure, relationships, flows, and frameworks.

**Architectural Clarity**
Layouts should feel deliberate, structured, and balanced. Use generous negative space. Allow content to breathe. Avoid visual clutter.

**Premium Restraint**
Less is more. Use minimal color palette. Favor typography over illustrations. Prefer clean lines over complex shapes. Quality over quantity.

**Data-Driven Storytelling**
Visuals must serve the narrative. Each diagram advances understanding. Charts reveal patterns. Process maps show transformation.

## Color Application Rules

### Color Palette

**Primary Palette**
- Forest Green: #0E3E1B - Primary actions, high-value elements, strategic emphasis
- Teal: #2A4E45 - Secondary elements, supporting data, mid-tier importance
- Slate: #697485 - Baseline data, neutral elements, low-tier information
- Beige: #F1EEEA - Light backgrounds, canvas, breathing room
- Charcoal: #12211D - Dark panels, contrast backgrounds, emphasis zones

**Accent Palette**
- Lavender: #CB9FD2 - Highlights, AI/intelligence features, premium callouts
- Coral: #FFC9C1 - Warm accents, dividers, subtle emphasis
- Cream: #FAF9F7 - Ultra-light backgrounds, layered surfaces
- Off-White: #FFFFFF - Pure backgrounds, cards, panels

**Text Palette**
- Primary Text: #212427 - Body copy, headlines, labels
- Light Text: #F1EEEA - Text on dark backgrounds
- Secondary Text: #697485 - Captions, annotations, metadata

### Color Usage Guidelines

**For Charts and Graphs:**

Single-Metric Charts:
- Use Forest Green (#0E3E1B) for primary data series
- Use gradient from Slate to Forest for progress indicators
- Use Teal (#2A4E45) for secondary comparisons

Multi-Metric Charts:
- Assign colors by importance: Forest > Teal > Slate > Lavender > Coral
- Never use more than 5 colors in one chart
- Maintain consistent color meaning across related charts

**For Diagrams:**

Flow Diagrams:
- Standard nodes: White background, Slate border
- Active/highlighted nodes: Forest Green border, light green background
- Premium nodes: Lavender border, lavender background with star
- Connecting lines: Teal or Slate, 2px weight

Grids and Frameworks:
- Quadrants: Use background tints at 5-10% opacity
- Borders: 2px solid, color matches category
- Highlighted zones: 20% background opacity, 3px border, shadow

Heat Maps:
- Low intensity: Slate (#697485)
- Medium intensity: Teal (#2A4E45)
- High intensity: Forest Green (#0E3E1B)
- Very high intensity: Forest Green with 20% background tint

**For Emphasis:**

High-Value Callouts:
- Border: 3px Forest Green or Lavender
- Background: 10% color tint
- Shadow: Subtle, 20% opacity, 20px blur

Medium-Value Callouts:
- Border: 2px Teal
- Background: 5% color tint
- Shadow: None or minimal

Baseline Information:
- Border: 1px Slate at 20% opacity
- Background: White or ultra-light beige
- Shadow: None

### Color Contrast Requirements

**For Accessibility:**
- Text on light backgrounds: Minimum 4.5:1 contrast ratio
- Text on dark backgrounds: Minimum 4.5:1 contrast ratio
- Data visualization elements: Minimum 3:1 against adjacent colors
- Interactive elements: Minimum 3:1 for focus states

**Tested Combinations:**
- #212427 on #F1EEEA: 16.2:1 ✓
- #F1EEEA on #12211D: 14.8:1 ✓
- #697485 on #F1EEEA: 5.2:1 ✓
- #0E3E1B on #F1EEEA: 8.9:1 ✓
- #FFFFFF on #0E3E1B: 12.1:1 ✓

## Layout Principles

### Grid System

**Column Grid:**
- Desktop: 12 columns
- Tablet: 8 columns
- Mobile: 4 columns
- Gutter: 24px (3 grid units)
- Margin: 80px desktop, 40px tablet, 24px mobile

**Spatial Grid (8pt System):**
All spacing increments in multiples of 8:
- 8px (1 unit) - Micro spacing
- 16px (2 units) - Tight spacing
- 24px (3 units) - Standard gap
- 32px (4 units) - Section spacing
- 40px (5 units) - Component padding
- 48px (6 units) - Large gaps
- 64px (8 units) - Section margins
- 80px (10 units) - Page margins
- 120px (15 units) - Section padding

### Composition Rules

**Visual Hierarchy:**
1. One dominant element per section
2. Clear focal point established through size, color, or position
3. Supporting elements scaled proportionally smaller
4. Tertiary elements subdued through color or size

**Negative Space:**
- Minimum 40% of each section should be empty space
- Allow content to breathe
- Create clear separation between concepts
- Use whitespace as a design element

**Alignment:**
- Left-align text blocks for readability
- Center-align headlines and statistics for emphasis
- Grid-align all diagram elements
- Maintain consistent margins throughout

**Balance:**
- Asymmetric balance preferred over perfect symmetry
- Visual weight distributed across composition
- Heavier elements balanced by negative space
- Color intensity balanced across layout

### Section Layouts

**Full-Width Hero:**
- Content max-width: 1200px
- Centered on page
- Padding: 120px vertical, 80px horizontal
- Two-column grid for content + visual

**Split Layout:**
- 50/50 or 60/40 column division
- Divider: 2px solid line or generous gap
- Left: Text/context
- Right: Visual/data
- Vertical center alignment

**Grid Layout:**
- 2×2 for frameworks
- 3×3 for capability stacks
- 4×1 for KPI strips
- 2×4 for comparison grids
- Equal spacing between cells

**Vertical Stack:**
- Single column, centered
- Max width: 700px for text, 1000px for visuals
- Consistent vertical rhythm
- Clear section breaks

**Horizontal Flow:**
- Timeline layouts
- Process flows
- Journey maps
- Equal-width stages or proportional to importance
- Connecting arrows or lines

## Diagram Design Patterns

### Donut Charts

**Structure:**
- Outer radius: 70-90px
- Inner radius: 40-50px (creates 30-40px ring)
- Stroke width: 40px
- Gap between segments: 2-3px visual separation

**Center Label:**
- Circle diameter: 80-100px
- Background: Solid color or subtle gradient
- Text: 12-14px, semibold
- Two lines maximum
- Centered alignment

**Legend:**
- Position: Below chart
- Items: Color dot + label + percentage
- Spacing: 12px vertical between items
- Alignment: Left-aligned or centered

**Color Application:**
- Start at top (12 o'clock position)
- Rotate clockwise
- Highest value or most important segment first
- Descending order of importance
- Use 3-5 colors maximum

**Use Cases:**
- Category distribution
- Maturity assessments
- Portfolio composition
- Market share
- Resource allocation

### Bar Charts

**Bar Structure:**
- Height: 12-16px for horizontal bars
- Corner radius: 4-8px (fully rounded ends)
- Spacing: 16-24px between bars
- Background track: 10% opacity of bar color

**Labels:**
- Category label: Left of bar, 14px
- Value label: Right of bar or inside bar end, 16px bold
- Percentage display: Adjacent to value
- Alignment: Baseline-aligned with bar

**Animation (Optional):**
- Bars fill from left to right
- Duration: 500-800ms
- Easing: Ease-out
- Stagger: 100ms delay between bars

**Color Application:**
- Single metric: One color throughout
- Comparative metrics: Two colors alternating
- Ranked data: Gradient from low (Slate) to high (Forest)
- Highlighted bar: Different color + shadow

**Use Cases:**
- Performance comparison
- Ranking visualization
- Progress tracking
- Survey results
- Metric benchmarking

### Flow Diagrams

**Node Design:**
- Shape: Rounded rectangle (12-16px radius) or circle
- Size: 80-120px width, 60-80px height
- Border: 2px solid
- Background: White or subtle tint
- Padding: 16px internal

**Node States:**
- Default: Slate border, white background
- Active: Forest Green border, 10% green tint
- Highlighted: Lavender border, shadow, star icon
- Completed: Checkmark, muted color

**Connectors:**
- Style: Solid line or arrow
- Weight: 2px
- Color: Teal or Slate
- Arrow style: Simple triangle, 8-12px
- Curve: Subtle bezier for organic flow

**Labels:**
- Inside node: 14px, semibold, centered
- Below node: 12px, regular, supporting text
- On connector: 11px, metadata

**Layout:**
- Horizontal: Left to right progression
- Vertical: Top to bottom hierarchy
- Circular: Cyclical processes
- Grid: Matrix relationships

**Use Cases:**
- User journeys
- Process flows
- Decision trees
- System architectures
- Value chains

### Progress Bars

**Bar Structure:**
- Height: 8-12px
- Corner radius: Fully rounded (9999px)
- Background: 10% opacity of progress color
- Progress fill: Solid color
- Width: Responsive to container

**Variants:**

Gradient Fill:
- Start color: Slate (#697485)
- Mid color: Teal (#2A4E45)
- End color: Forest (#0E3E1B)
- Position gradient based on percentage

Segmented:
- Divide into equal sections
- Fill completed segments
- Partial fill for current segment
- Gray unfilled segments

Stacked:
- Multiple metrics in one bar
- Different colors for each
- Total adds to 100%
- Legend required

**Labels:**
- Start label: Left side, category name
- End label: Right side, percentage or value
- Milestone markers: Above bar at key points

**Use Cases:**
- Goal tracking
- Maturity assessment
- Skill levels
- Resource allocation
- Time progression

### Heat Maps

**Grid Structure:**
- Cell size: 80-120px square or rectangular
- Gap: 8-12px between cells
- Border: 1px or 2px on cells
- Corner radius: 8-12px

**Color Intensity:**
- Low: Slate (#697485) at 20% opacity
- Medium-Low: Slate at 40% opacity
- Medium: Teal (#2A4E45) at 60% opacity
- Medium-High: Forest (#0E3E1B) at 40% opacity
- High: Forest (#0E3E1B) at 80% opacity
- Very High: Forest (#0E3E1B) solid

**Cell Content:**
- Label: 12px, semibold, top or centered
- Value: 11px, regular, secondary position
- Icon (optional): 24px, centered

**Legend:**
- Position: Below or beside grid
- Style: Gradient bar or discrete boxes
- Labels: Low, Medium, High or specific values

**Use Cases:**
- Value assessment
- Priority matrices
- Performance grids
- Feature comparison
- Risk analysis

### Quadrant Grids

**Structure:**
- 2×2 grid, equal quadrants
- Dimensions: 400-600px square total
- Gap: 8-16px between quadrants
- Aspect ratio: 1:1

**Quadrant Design:**
- Background: 5-10% color tint
- Border: 2px solid, color-coded
- Corner radius: 12-16px
- Padding: 24-32px internal

**Axis Labels:**
- X-axis: Below grid, centered, 12px
- Y-axis: Left side, rotated 90°, 12px
- Arrow indicators: Optional, subtle
- Color: Slate (#697485)

**Quadrant Labels:**
- Position: Centered in quadrant
- Size: 14-16px
- Weight: Semibold
- Color: Matches quadrant category
- Supporting text: 12px below

**Highlight Treatment:**
- Target quadrant: Darker border, shadow
- Icon: Star or checkmark, 24px
- Background: 20% opacity vs 5% for others

**Use Cases:**
- Competitive positioning
- Priority assessment
- Strategic frameworks
- Portfolio analysis
- Capability mapping

### Timeline Visualizations

**Structure:**

Horizontal Timeline:
- Length: Full width, responsive
- Central line: 2px solid, Slate
- Nodes: Circles or rounded rectangles
- Spacing: Equal or proportional to time

Vertical Timeline:
- Height: Variable based on content
- Left line: 2px solid, Slate
- Nodes: Left-aligned circles
- Content: Right side of line

**Node Design:**
- Shape: Circle 40-60px diameter or rounded square
- Border: 2px solid, color-coded
- Background: White or color tint
- Icon or number: Centered, 16-20px

**Node States:**
- Past: Muted color, 50% opacity
- Current: Full color, larger size, shadow
- Future: Light color, 30% opacity, dashed line

**Labels:**
- Date/period: Above or beside node, 12px
- Title: 16px, semibold, primary position
- Description: 14px, regular, below title
- Max width: 200-300px per item

**Connecting Lines:**
- Between nodes: 2px solid or dashed
- Color: Slate or fades with node state
- Style: Straight or subtle curve

**Use Cases:**
- Historical progression
- Roadmaps
- Evolution narratives
- Phase planning
- Milestone tracking

### Capability Stacks

**Structure:**

Layered Model:
- 3-5 horizontal layers
- Stack vertically
- Equal or proportional heights
- Clear separation between layers

Center-Out Model:
- Central circle: Core strategy/capability
- Surrounding layers: Supporting elements
- Concentric or radial arrangement

Grid Model:
- 3×3 or similar matrix
- Central element larger or highlighted
- Surrounding cells supporting

**Layer Design:**
- Height: 80-120px per layer
- Padding: 16-24px
- Border: 1px between layers or clear gap
- Background: Gradient from light (top) to dark (bottom)

**Element Design:**
- Box per capability: 120-200px wide
- Corner radius: 8-12px
- Border: 2px, color-coded by layer
- Background: White or subtle tint
- Content: Centered text, 12-14px

**Color Coding:**
- Top layer: Forest Green (outcomes)
- Middle layer: Teal (technology)
- Bottom layer: Slate (enablers)
- Consistent across similar diagrams

**Labels:**
- Layer label: Left or top, 16px semibold
- Element label: Inside box, 12-14px
- Icons: 24px, optional, top of box

**Use Cases:**
- Technology stacks
- Organizational models
- Capability frameworks
- Operating architectures
- Strategic layers

### Comparison Panels

**Structure:**

Side-by-Side:
- Two equal columns
- Vertical divider: 2px solid, Coral (#FFC9C1)
- Padding: 40-60px per side
- Alignment: Top-aligned content

Before/After:
- Left: "Before" state, muted colors
- Right: "After" state, vibrant colors
- Clear visual distinction
- Parallel structure for easy comparison

**Panel Design:**
- Background: Light for "before", subtle tint for "after"
- Border: Optional, 1px Slate
- Padding: 40px internal
- Corner radius: 12-16px

**Content Structure:**
- Title: 20px, semibold, top of panel
- Items: List format, 16px
- Icons: 20-24px, left of text
- Spacing: 16-20px between items

**Improvement Indicators:**
- Arrow icons: Up/down, 16px
- Percentage change: Bold, colored
- Visual highlight: Green for positive, neutral for negative
- Position: Right side of metric

**Use Cases:**
- Before/after scenarios
- Traditional vs. AI-driven
- Current vs. target state
- Option comparison
- Competitive analysis

## Data Visualization Standards

### Chart Hierarchy

**Primary Charts:**
Communicate main insights, largest visual real estate, prominent colors

**Secondary Charts:**
Support primary insights, medium size, complementary colors

**Tertiary Charts:**
Provide context, smaller size, muted colors

### Data Point Treatments

**Highlighting Important Data:**
- Size increase: 20-30% larger
- Color: Forest Green or Lavender
- Shadow: Subtle drop shadow
- Annotation: Callout line and label
- Icon: Star or checkmark

**Trend Indicators:**
- Upward trend: Arrow up, green
- Downward trend: Arrow down, neutral (not red)
- Stable trend: Horizontal line, gray
- Position: Adjacent to data point

**Value Labels:**
- Always show key values
- Format: Consistent (%, $, whole numbers)
- Position: Above bar, inside donut, end of line
- Size: 14-16px, bold
- Color: Match data or contrast for readability

### Chart Responsiveness

**Desktop (1440px+):**
- Side-by-side charts
- Full detail visible
- Legends beside charts
- Annotations visible

**Tablet (768-1024px):**
- Charts stack or scale
- Legends below charts
- Simplified annotations
- Touch-friendly targets

**Mobile (375-768px):**
- Single column
- Simplified charts
- Essential data only
- Swipe for additional views

## Accessibility Guidelines

### Color Accessibility

**Never Rely on Color Alone:**
- Use text labels
- Add patterns or textures
- Include icons or symbols
- Provide alternative indicators

**High Contrast:**
- Text minimum 4.5:1 ratio
- Interactive elements minimum 3:1
- Charts minimum 3:1 between adjacent

**Color Blindness Considerations:**
- Avoid red-green combinations
- Use tested palette
- Provide alternative visual cues
- Test with simulation tools

### Interaction Accessibility

**Touch Targets:**
- Minimum 44×44px
- Adequate spacing between
- Clear active states
- No overlapping targets

**Focus States:**
- Visible outline or border
- Minimum 2px
- High contrast color
- Never remove focus indicators

**Keyboard Navigation:**
- Logical tab order
- Skip links for long content
- All interactive elements reachable
- Clear focus indicator

### Content Accessibility

**Text Alternatives:**
- Alt text for meaningful images
- Captions for charts
- Text summaries of data
- ARIA labels where appropriate

**Structure:**
- Semantic HTML
- Proper heading hierarchy
- Landmarks for navigation
- Descriptive link text

**Motion:**
- Respect prefers-reduced-motion
- No auto-playing animations
- Pause controls for movement
- Alternative static views

## Export and Production Specifications

### File Formats

**For Web:**
- Diagrams: SVG (vector, scalable)
- Charts: SVG or Canvas
- Icons: SVG inline
- Images: WebP with PNG fallback

**For Print:**
- Diagrams: PDF or high-res PNG (300dpi)
- Charts: Vector PDF preferred
- Full layouts: PDF with embedded fonts
- Colors: CMYK conversion verified

**For Presentations:**
- Individual diagrams: PNG (2x resolution)
- Full slides: PDF or PPTX
- Animations: MP4 or GIF for key transitions

### Resolution Standards

**Screen:**
- 1x: Base resolution
- 2x: Retina displays
- SVG: Preferred for infinite scaling

**Print:**
- 300dpi minimum
- Vector when possible
- Bleed: 3mm on printed materials
- Safe zone: 5mm from edge

### Naming Conventions

**File Names:**
- descriptive-name-type-version.extension
- Example: consumer-intelligence-donut-v2.svg
- Lowercase with hyphens
- No spaces or special characters

**Asset Organization:**
- /diagrams/ - Strategic frameworks
- /charts/ - Data visualizations
- /icons/ - Functional icons
- /layouts/ - Full section layouts

## Design Decision Framework

### When to Use Which Visual

**Use Donut Charts When:**
- Showing part-to-whole relationships
- 3-5 categories maximum
- Percentages add to 100%
- Comparing two distributions
- Maturity or composition analysis

**Use Bar Charts When:**
- Comparing quantities across categories
- Showing rankings
- Displaying progress over discrete items
- More than 5 data points
- Need precise value comparison

**Use Line Charts When:**
- Showing trends over time
- Continuous data
- Multiple series comparison
- Forecasting or projections
- Emphasizing change rate

**Use Flow Diagrams When:**
- Illustrating processes
- Showing user journeys
- Mapping decision trees
- Explaining transformations
- Depicting sequential stages

**Use Heat Maps When:**
- Comparing values across two dimensions
- Showing intensity or priority
- Matrix frameworks
- Pattern recognition
- Dense data in compact space

**Use Quadrant Grids When:**
- Strategic positioning
- Portfolio analysis
- Priority matrices
- Capability assessment
- Comparing on two axes

**Use Timeline When:**
- Historical progression
- Roadmap planning
- Evolution narrative
- Phase-based projects
- Before/during/after scenarios

**Use Progress Bars When:**
- Goal tracking
- Completion status
- Skill or maturity levels
- Resource allocation
- Simple comparisons

**Use KPI Panels When:**
- Emphasizing key metrics
- Dashboard views

- Quick insights
- Statistical highlights

## Common Mistakes to Avoid

### Visual Mistakes

**Too Much Color:**
Limit to 3-4 colors per diagram. More creates chaos and dilutes meaning.

**Inconsistent Styling:**
Same element types must look identical. Don't vary styles arbitrarily.

**Poor Contrast:**
Ensure readability. Test in grayscale. Avoid low-contrast combinations.

**Misaligned Elements:**
Use grids. Align to baselines. Precision matters for professional appearance.

**Cluttered Layouts:**
Remove unnecessary elements. Embrace whitespace. Less is more.

**Decorative Elements:**
Every element must have purpose. No decoration for decoration's sake.

### Data Mistakes

**Misleading Scales:**
Start bar charts at zero. Use consistent scales. Don't distort data visually.

**Too Much Data:**
Focus on key insights. Remove noise. Simplify complex datasets.

**Missing Context:**
Provide comparisons. Show benchmarks. Label axes clearly.

**Unclear Labels:**
Be specific. Avoid jargon. Define abbreviations.

**No Hierarchy:**
Emphasize what matters. De-emphasize supporting data. Guide the eye.

### Content Mistakes

**Vague Headlines:**
Be specific about the insight. Avoid generic titles.

**Inconsistent Terminology:**
Choose terms and stick with them. Create a glossary if needed.

**No Clear Takeaway:**
Every visual should answer: "So what?" Make the insight obvious.

**Wrong Visual Type:**
Match diagram to data type and story. Don't force square pegs in round holes.

## Iteration and Refinement

### Design Review Process

**First Pass: Structure**
- Is the visual type appropriate?
- Does layout support hierarchy?
- Is the flow logical?

**Second Pass: Details**
- Are colors applied correctly?
- Is typography consistent?
- Are spacing and alignment precise?

**Third Pass: Content**
- Is data accurate?
- Are labels clear?
- Is the insight obvious?

**Fourth Pass: Polish**
- Remove anything unnecessary
- Refine rough edges
- Test at multiple sizes
- Verify accessibility

### Testing Criteria

**Glance Test:**
Can someone grasp the main point in 3 seconds?

**Context-Free Test:**
Does it make sense without surrounding text?

**Accessibility Test:**
Can it be understood without color?

**Scale Test:**
Does it work at thumbnail and full size?

**Reusability Test:**
Can the pattern be applied to similar data?
